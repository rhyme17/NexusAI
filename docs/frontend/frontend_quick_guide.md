# NexusAI 前端用户快速上手（业务简版）

更新日期：2026-04-08
适用对象：业务同学、非研发内部用户

## 1. 你能用它做什么

- 创建一个协作任务（例如：技术调研、方案草案、风险评估）
- 观察多 Agent 协作过程（认领、交接、执行、失败、重试）
- 查看最终输出结果与执行解释

## 2. 进入系统后先做两件事

1. 打开 `设置 /settings`，设置 `用户 AI Key`（用于真实执行）
2. 如果后端启用了鉴权，在 `设置 /settings` 中再设置 `后端 API Key`

说明：
- 没有用户 AI Key 时，真实执行可能失败
- 当前默认不自动回退到模拟结果（避免误判）

## 3. 3-5 分钟完成一次完整流程

1. 打开 `任务台`（`/tasks`）
2. 输入任务目标并创建任务
3. 系统会自动进入该任务工作区（`/tasks/{taskId}`）
4. 在 `执行` 页签点击 `预览执行`
5. 点击 `开始执行`
6. 在 `概览` 看 `结果`，在 `事件流` 看全过程

## 4. 页面速览

- `总览 /`：看系统状态、焦点任务、最近任务
- `任务台 /tasks`：创建与筛选任务
- `单任务 /tasks/{taskId}`：执行控制、事件、结果、流程图
- `智能体 /agents`：角色、状态、技能、状态管理
- `设置 /settings`：语言、用户 Key、后端 Key、Agent 注册、服务器数据操作

## 5. 结果判断（避免误读）

- 真实执行：结果里通常会出现 `mode=real`
- 模拟执行：由 `Simulate Success/Failure` 触发，仅用于联调
- 若执行失败：先看 `执行提示` 与 `事件流` 再决定是否重试
- 需要落盘结果时：在任务详情 `概览` 可直接 `导出 .md / .txt` 或 `复制为 Markdown`

## 6. 常见问题

### Q1. 点了“开始执行”但没有结果

优先检查：
- 后端是否在线（`/health`）
- 用户 AI Key 是否已设置
- 是否先做了 `预览执行`

### Q2. 结果看起来像状态摘要，不像报告正文

建议把任务写得更具体，并在目标里明确要求输出格式，例如：
- 背景
- 关键结论
- 风险与建议
- 可执行计划

### Q3. 我只想联调流程，不关心真实模型

可使用：
- `Simulate Failure` + `Retry`
- `Simulate Success`

## 7. 推荐的任务描述模板（可直接复制）

```text
请调研“区块链在信息安全中的应用”，输出一份 Markdown 报告，至少包含：
1) 背景与范围
2) 主要应用场景与案例
3) 技术优缺点与风险
4) 结论与可执行建议
```
