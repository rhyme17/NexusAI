# NexusAI 用户使用文档（面向业务用户）

更新日期：2026-04-10  
适用对象：业务同学、产品经理、演示人员、非研发用户

## 1. 这份文档能帮你做什么

你可以按本文快速完成一条完整闭环：

- 创建一个任务目标
- 让系统执行多 Agent 协作
- 查看过程与结果
- 导出结果用于汇报或二次编辑

如果你是研发/测试/运维，请优先查看：`docs/frontend/frontend_operations_manual.md`。

## 2. 开始前准备

请先确认：

- 可以访问前端页面（默认 `http://localhost:3000`）
- 后端服务在线（若离线，页面会出现连接失败提示）
- 你已拿到可用的用户 AI Key（用于真实执行）

说明：

- 用户 AI Key 仅用于触发真实模型执行
- 如果你的环境启用了后端 API 鉴权，通常由管理员统一配置；普通用户按团队说明操作即可

## 3. 第一次使用（建议 2 分钟完成）

1. 打开 `设置 /settings`
2. 在 `用户 AI Key` 区域填写并保存你的 key
3. 返回 `任务台 /tasks`

完成后，你就可以执行真实任务。

## 4. 标准使用流程（3-5 分钟）

1. 打开 `任务台 /tasks`
2. 在创建区域输入任务目标（objective）并选择优先级
3. 提交后进入任务详情页 `单任务 /tasks/{taskId}`
4. 在 `执行` 页签先点击 `预览执行`
5. 确认预览信息后点击 `开始执行`
6. 在 `概览` 查看结果，在 `事件流` 查看全过程

推荐先做一次小任务，确认链路正常后再执行复杂任务。

## 5. 页面导航说明

- `总览 /`：看系统健康、关键统计与最近任务
- `任务台 /tasks`：创建任务、筛选任务、进入任务详情
- `单任务 /tasks/{taskId}`：执行任务、看结果、看事件、看流程图
- `智能体 /agents`：查看当前可用智能体及其状态
- `设置 /settings`：语言切换、用户 AI Key、基础连接相关设置

## 6. 如何判断执行是否成功

你可以用下面 4 条快速判断：

- 任务状态到达 `completed`
- `概览` 中出现可读结果内容
- `事件流` 中看到执行完成相关事件
- 结果区可正常导出 `.md/.txt` 或复制 Markdown

如状态为 `failed`，先查看执行提示，再决定是否重试。

## 7. 结果导出与复用

在任务详情 `概览` 页签可直接：

- 导出为 `.md`
- 导出为 `.txt`
- 复制为 Markdown

建议对外发送前补充你自己的业务上下文与结论。

## 8. 常见问题

### Q1：点击开始执行后没有结果

优先检查：

- 后端是否在线
- 用户 AI Key 是否已配置
- 是否先执行了 `预览执行`

### Q2：输出太泛，不够可执行

优化方法：

- 把任务写成结构化要求
- 在目标里明确输出格式（例如分章节）
- 指定你希望的结果类型（结论、风险、行动项）

### Q3：只想演示流程，不调用真实模型

可使用任务页中的模拟按钮（如 `Simulate Success/Failure`）进行联调演示。模拟结果不代表真实模型质量。

## 9. 推荐任务模板（可直接复制）

```text
请调研“区块链在信息安全中的应用”，输出一份 Markdown 报告，至少包含：
1) 背景与范围
2) 主要应用场景与案例
3) 技术优缺点与风险
4) 结论与可执行建议（按优先级）
```

## 10. 安全与使用注意事项

- 不要在公共设备长期保存个人 key
- 不要把 key 截图、发群或写入公开文档
- 如怀疑 key 泄露，请立即撤销并更换
- 模拟执行仅用于演示和联调，不用于业务结论
